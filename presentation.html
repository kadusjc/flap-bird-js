🎯 Project Overview

A Flappy Bird-style game running entirely in the browser, enhanced with a YOLOv5 neural network that performs real-time object detection on the game canvas.

Goal: Demonstrate how machine learning models can run client-side in a browser using TensorFlow.js, with zero backend infrastructure.
  • Pure HTML5 Canvas + vanilla JavaScript (ES Modules)
  • AI runs in a Web Worker (separate thread)
  • Model: YOLOv5n (nano) — the smallest & fastest YOLOv5 variant
  • No server, no API calls — everything runs on the user's device

🔄 The Game Loop

The heartbeat of every real-time game. Runs ~60 times per second.

// main.js
function loop() {
  game.update()   // Logic: physics, collisions
  game.draw()     // Render: redraw everything
  requestAnimationFrame(loop)  // Schedule next
}
loop()  // Start!
requestAnimationFrame
Browser API that calls your function right before the next screen repaint. Automatically pauses when the tab is hidden, saving CPU/GPU.

At 60 fps each frame is ~16.67 ms. All update + draw logic must finish within this budget.

🎮 Game.js — The Orchestrator

Responsibilities

  • Manages Player, Obstacles[], LevelManager, Evasion
  • Spawns obstacles at intervals based on current level
  • Runs collision detection every frame
  • Draws all entities + HUD (score, level)
  • Sends canvas snapshots to the Worker every 200 ms
  • Handles game over & win condition (score ≥ 80)
  • Delegates AI evasion to Evasion.js
  • Level-up announcements with colored fade-out text

Constructor

🎮 Game.js — update()

update() {
  if (!this.running) return
  this.frame++
  // AI evasion: if fixedY is set, lock player; otherwise normal physics
  if (this.fixedY !== null) {
    this.evasion.revalidateFixedY()       // Re-check safety each frame
    this.player.y = this.fixedY; this.player.vy = 0
  } else {
    this.player.update()
    this.evasion.emergencyEvade()          // Emergency: frame-by-frame check
  }
  if (this.frame % this.levelManager.spawnRate() === 0) this.spawn()

  for (const o of this.obstacles) o.update(this.levelManager.speed())
  this.evasion.lastResortEvade()          // Last resort using YOLO predictions

  for (let i = this.obstacles.length - 1; i >= 0; i--) {
    let o = this.obstacles[i]
    if (collide(this.player, o)) { this.gameOver(); return }
    if (o.x + o.w < 0) {
      this.obstacles.splice(i, 1); this.score++
      if (this.levelManager.update(this.score))
        this.levelAnnounce = { level: this.levelManager.level, startTime: performance.now() }
      if (this.score >= 80) { this.youWin(); return }
    }
  }
}
Why iterate backwards? splice(i, 1) shifts indices. Iterating backwards (i--) prevents skipping elements.
3-tier evasion: revalidateFixedY() keeps lock safe, emergencyEvade() reacts frame-by-frame, and lastResortEvade() is the final safety net — all via YOLO predictions.

🎮 Game.js — draw()

Redraws the entire scene every frame. Order matters — later draws appear on top.

draw() {
  this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height) // 1. Clear
  if (this.bg.complete)                                           // 2. Background
    this.ctx.drawImage(this.bg, 0, 0, this.canvas.width, this.canvas.height)
  this.player.draw(this.ctx)                     // 3. Player
  for (let o of this.obstacles) o.draw(this.ctx)  // 4. Obstacles (with rotation)
  this.ctx.fillStyle = "white"                   // 5. HUD
  this.ctx.font = "20px Arial"
  this.ctx.fillText("Score: " + this.score, 10, 25)
  this.ctx.fillText("Level: " + this.levelManager.level, 10, 50)
  // this.drawPredictions()                       // 6. YOLO bboxes (debug toggle)
  this.drawLevelAnnounce()                        // 7. Level-up announcement
  if (!this.running) {                            // 8. End-game overlay
    if (this.won) this.drawYouWin()               //    🏆 Gold "YOU WIN"
    else this.drawGameOver()                      //    🔴 Red "GAME OVER"
  }
}
Canvas 2D is immediate-mode — nothing retained between frames. Includes optional YOLO bbox overlay (commented out), level-up announcements with colored fade-out, and win/lose screens.

🎮 Game.js — Worker Communication

setupWorker() {
  this.worker.onmessage = (e) => {
    const { type } = e.data
    if (type === 'ready') console.log('[Game] Worker pronto')
    if (type === 'action' && e.data.action === 'jump') this.player.jump()
    if (type === 'predictionResult') {
      this.predictions.push({ label: e.data.label, confidence: e.data.confidence,
        bbox: e.data.bbox, snapshotTime: e.data.sentAt || performance.now(),
        timestamp: performance.now() })
      this.predictions = this.predictions.filter(p => performance.now() - p.timestamp < 1000)
      this.evasion.makeAirplaneMoveAwayObstacles(e.data)  // AI evasion!
    }
  }
  // Send canvas snapshot to Worker every 200ms
  this.snapshotInterval = setInterval(async () => {
    if (!this.running) return
    const sentAt = performance.now()
    const bitmap = await createImageBitmap(this.canvas)
    this.worker.postMessage({ type: 'predict', image: bitmap, sentAt }, [bitmap])
  }, 200)
}

✈️ Player.js — The Bird

class Player {
  constructor() {
    this.x = 80       // Fixed horizontal pos
    this.y = 200      // Starting Y
    this.w = 60; this.h = 40  // Size
    this.vy = 0       // Vertical velocity
    this.sprite = new Image()
    this.sprite.src = "./assets/player.png"
  }

  jump() { this.vy = -10 }

  update() {
    this.vy += 0.5     // Gravity
    this.y += this.vy   // Apply velocity
    if (this.y < 0) this.y = 0
    if (this.y > 350) { this.y = 350; this.vy = 0 }
  }

  draw(ctx) {
    if (this.sprite.complete)
      ctx.drawImage(this.sprite,
        this.x, this.y, this.w, this.h)
  }
}

Physics Model

Euler Integration (simplest physics): velocity += gravity → acceleration; position += velocity → movement. Each frame, vy increases by 0.5 (gravity). jump() sets vy = -10 (upward).
Canvas coordinate system: Y=0 is the top. Y increases downward.
So vy = -10 means "go up".

🪨 Obstacle.js

class Obstacle {
  constructor(canvas) {
    // Random square size 35-90px (meteor)
    const size = Math.random() * 55 + 35
    this.w = size
    this.h = size  // square = round without distortion

    this.x = canvas.width  // spawn at right edge
    this.y = Math.random()
            * (canvas.height - this.h)

    this.sprite = new Image()
    this.sprite.src = "./assets/obstacle.png"

    // Rotation animation
    this.angle = Math.random() * Math.PI * 2
    this.rotationSpeed = (Math.random()
      * 0.04 + 0.02)
      * (Math.random() < 0.5 ? 1 : -1)
  }

  update(speed) {
    this.x -= speed
    this.angle += this.rotationSpeed
  }

  draw(ctx) {
    ctx.save()
    ctx.translate(cx, cy)  // center
    ctx.rotate(this.angle)
    ctx.drawImage(sprite, ...)
    ctx.restore()
  }
}

Obstacle Lifecycle

  1. Spawn — Created at x = canvas.width (right edge)
  2. Move — Each frame: x -= speed
  3. Rotate — Each frame: angle += rotationSpeed
  4. Collide? — Checked by collide(player, obstacle)
  5. Off-screen — When x + w < 0 → removed, score++
Design choice: Obstacles are square (35–90px) with random rotation, simulating spinning meteors. save/translate/rotate/restore draws the sprite rotated around its center.

💥 Collision.js — AABB Detection

export default function collide(a, b) {
  return (
    a.x < b.x + b.w &&  // A left < B right
    a.x + a.w > b.x &&  // A right > B left
    a.y < b.y + b.h &&  // A top < B bottom
    a.y + a.h > b.y     // A bottom > B top
  )
}
If all four conditions are true → overlap → collision!

AABB — Axis-Aligned Bounding Box

  • Axis-Aligned = edges parallel to X/Y axes
  • Bounding Box = smallest enclosing rectangle
  • O(1) — just 4 comparisons
  • Used in Unity, Unreal, Godot, and virtually every 2D engine

For rotated objects → OBB or SAT (Separating Axis Theorem).

🔍 What is YOLO? — History

  • 2015 — YOLOv1 by Joseph Redmon: "You Only Look Once"
  • 2016-2020 — v2, v3 by Redmon; v4 by Bochkovskiy et al.
  • 2021 — YOLOv5 by Ultralytics (Glenn Jocher). First YOLO in PyTorch.
  • 2023-2025 — YOLOv8, v9, v10, v11 by various teams
Unlike R-CNN (multi-pass), YOLO processes the entire image in a single forward pass.

How It Works

  1. Divide image into S×S grid
  2. Each cell predicts B boxes + confidence
  3. Each cell predicts C class probabilities
  4. NMS removes overlapping boxes

⚙️ Web Workers — Why & How

The Problem

JavaScript is single-threaded. Running a neural network (50-200ms) on the main thread would freeze the game loop.

The Solution

Web Workers run JS in a separate OS thread, communicating via postMessage().

Workers cannot: access DOM, window, or canvas. Messages only.

Communication Flow

Main Thread          Web Worker
    │──── predict ────▶  │
    │    (ImageBitmap)    │ ← runs YOLOv5
    │◀── prediction ────  │
    │    (bounding boxes) │
    │──── gameState ──▶  │
    │    (positions)      │ ← decides action
    │◀── action ────────  │
    │    (jump/none)      │

🧠 GameWorker.js — Model Loading

importScripts('https://cdn.jsdelivr.net/npm/@tensorflow/tfjs@latest')
const MODEL_PATH = '../machine-learning/yolov5n_web_model/model.json'
const LABELS_PATH = '../machine-learning/yolov5n_web_model/labels.json'

async function loadModelYOLOv5AndItsLabels() {
  await tf.ready()                                   // 1. Init TF backend
  _labels = await (await fetch(LABELS_PATH)).json()  // 2. Load class names
  _model = await tf.loadGraphModel(MODEL_PATH)       // 3. Load model
  // 4. Warmup — compile WebGL shaders
  const dummyInput = tf.ones(_model.inputs[0].shape)
  await _model.executeAsync(dummyInput)
  tf.dispose(dummyInput)
  postMessage({ type: 'model-loaded' })              // 5. Signal ready
}
Warmup: First inference is slow (WebGL compiles shaders + allocates GPU buffers). A dummy run at startup makes real predictions fast.

🧠 GameWorker.js — Image Preprocessing

function createAndFormatImageTensor(image) {
  return tf.tidy(() => {                                // Auto-dispose intermediates
    const tensorImage = tf.browser.fromPixels(image)    // [400, 800, 3] H×W×RGB
    return tf.image
      .resizeBilinear(tensorImage, [640, 640])          // [640, 640, 3]
      .div(255)                                         // Normalize to [0, 1]
      .expandDims(0)                                    // [1, 640, 640, 3]
  })
}
ImageBitmap
800×400 → fromPixels
[400,800,3] → resize
[640,640,3] → ÷ 255
0–1 → expandDims
[1,640,640,3]
tf.tidy() auto-disposes all intermediate tensors in the callback, except the returned one. Essential for preventing GPU memory leaks.

🧠 GameWorker.js — Running Inference

async function runYoloModelOverTensorAndGetDetections(tensor) {
  const output = await _model.executeAsync(tensor)     // 1. Run model
  tf.dispose(tensor)                                   // 2. Free input

  const [boxes, scores, classes, validDetections] = output.slice(0, 4)

  const [boxesData, scoresData, classesData, validData] = // 4. GPU → CPU
    await Promise.all([boxes.data(), scores.data(), classes.data(), validDetections.data()])

  output.forEach(t => t.dispose())                     // 5. Free outputs
  return { boxes: boxesData, scores: scoresData, classes: classesData,
           numDetections: validData[0] }
}
Model outputs (YOLOv5): boxes [x1,y1,x2,y2] · scores confidence 0–1 · classes integer index → labels.json · numDetections valid count (avoids iterating garbage slots)

🧠 GameWorker.js — Processing Predictions

// Generator function — yields predictions one at a time
function* processPrediction({ boxes, scores, classes, numDetections }, width, height) {
  const limit = numDetections ?? scores.length
  for (let index = 0; index < limit; index++) {
    if (scores[index] < CLASS_THRESHOLD) continue       // Filter low confidence
    const label = _labels[classes[index]]
    if (!['airplane', 'clock', 'stop sign', 'kite'].includes(label.toLowerCase())) continue

    let [x1, y1, x2, y2] = boxes.slice(index * 4, (index + 1) * 4)
    x1 *= width; x2 *= width; y1 *= height; y2 *= height // Normalize to pixels

    yield { label, confidence: scores[index],
            bbox: [x1, y1, x2, y2],
            center: [(x1+x2)/2, (y1+y2)/2] }
  }
}
Generator Functions (function*)
A special JavaScript function that can pause and resume execution. Uses yield to emit values lazily, one at a time. No intermediate array is created, saving memory.
CLASS_THRESHOLD = 0.4
Only detections with confidence ≥ 40% are kept. This filters out noise and false positives. In production, you'd tune this based on precision/recall requirements.

🧠 GameWorker.js — Message Handler

self.postMessage({ type: 'ready' })        // Signal: worker is alive

self.onmessage = async (e) => {
  const { type } = e.data
  if (type === 'predict') return handlePredictEvent(e)
}

async function handlePredictEvent(e) {
  if (!_model) return
  const image = e.data.image
  const { width, height } = image
  const input = createAndFormatImageTensor(image)
  image.close()  // Free native ImageBitmap memory
  try {
    const results = await runYoloModelOverTensorAndGetDetections(input)
    for (const prediction of processPrediction(results, width, height))
      postMessage({ type: 'predictionResult', ...prediction, sentAt: e.data.sentAt })
  } catch (err) {
    tf.dispose(input)
    console.error('[Worker] Prediction error:', err)
  }
}

loadModelYOLOv5AndItsLabels()
Key improvements: image.close() frees native bitmap memory. sentAt timestamp enables latency compensation in Evasion.js. handlePredictEvent isolates predict logic with proper error handling.

🎨 Key Design Decisions

✅ Good choices
  • Web Worker — main thread stays at 60 fps
  • Transferable ImageBitmap — zero-copy
  • YOLOv5n — smallest model for real-time
  • tf.tidy() + dispose() — GPU memory mgmt
  • Generator — lazy, memory-efficient
  • ES Modules — no build step
  • Evasion.js — dedicated class, 3-tier YOLO-only evasion
  • Latency compensation — sentAt timestamp corrects predictions
🔮 Future opportunities
  • Fine-tune YOLOv5 — retrain the model on actual game sprites so it recognizes "player" and "obstacle" instead of guessing "clock" or "kite"
  • OffscreenCanvas — render directly inside the Worker thread, eliminating main-thread canvas access entirely
  • Reinforcement Learning — replace hand-coded rules with a self-learning agent (Q-learning / NEAT) that improves by playing thousands of games

🤖 How The AI Plays

The full AI pipeline is implemented and plays autonomously:

Vision Pipeline

// Every 200ms:
// 1. Canvas → ImageBitmap → Worker
// 2. Worker runs YOLOv5n inference
// 3. Detections filtered (clock/kite/...)
// 4. postMessage({type:'predictionResult',
//    bbox, sentAt}) back to main thread
// 5. Evasion.js compensates latency:
//    obs.x -= speed * 60 * elapsed

3-Tier Evasion

// Tier 1: makeAirplaneMoveAwayObstacles()
//   Filters 'clock'/'stop sign' labels
//   Non-obstacle labels → ignored (no release)
//   → calcSafeY → set fixedY

// Tier 2: emergencyEvade()
//   Speed-scaled: 120 + speed*15 px
//   L8 (speed 7): 225px reaction range

// Tier 3: lastResortEvade()
//   Speed-scaled: 40 + speed*8 px
//   L8 (speed 7): 96px last-chance range
All 3 tiers use only YOLO predictions via getObstaclePredictions(). All distances scale with speed for high-level reactivity.

⚡ Performance Considerations

MetricValueNotes
Game loop target~16.67 ms (60 fps)requestAnimationFrame syncs to monitor
YOLO inference~50–200 msGPU-dependent; runs in Worker (non-blocking)
Frame capture200 ms (5 fps)Balance: AI responsiveness vs CPU load
Model size~4 MBDownloaded once, cacheable
Input tensor640×640×3~1.2M float32 values per frame
Why 200ms? YOLOv5 at 60 fps would overwhelm most GPUs. 5 fps gives the AI enough updates to react while keeping the game smooth.

📝 Summary

  • YOLO-only evasion — AI uses only YOLO predictions with latency compensation, no direct obstacle access
  • Zero backend — everything runs client-side in the browser
